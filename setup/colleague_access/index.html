    Colleague Access Setup¶

    This guide helps you set up database access for your colleagues working on the Event Discovery project.

    🎯 Overview¶

    When working with MongoDB Atlas in a team, you need to grant database access to your colleagues. This guide provides multiple methods to accomplish this securely.

    🚀 Quick Setup (Recommended)¶

    Step 1: Gather Colleague Information¶

    Create a file colleague_info.txt with the following format:

    # Colleague Information
Name: John Doe
Email: john.doe@company.com
IP Address: 192.168.1.100
Role: Developer
Access Level: readWrite

    Step 2: Run the Setup Script¶

    # Use the automated setup script
python colleague_setup.py

    📝 Manual Setup Method¶

    Step 1: Access MongoDB Atlas Console¶

    1. Login to MongoDB Atlas
    2. Select your project
    3. Navigate to the appropriate section for each task

    Step 2: Create Database User¶

    sequenceDiagram
    participant You
    participant Atlas as MongoDB Atlas
    participant Colleague

    You->>Atlas: Navigate to Database Access
    You->>Atlas: Click "Add New Database User"
    You->>Atlas: Enter colleague credentials
    You->>Atlas: Set permissions
    Atlas-->>You: User created successfully
    You->>Colleague: Share credentials securely

    Detailed Steps:

    1. Navigate to Database Access in the left sidebar
    2. Click Add New Database User
    3. Fill in the form: 
      Authentication Method: Password
Username: colleague_username
Password: [Generate secure password]
Database User Privileges: 
  - Read and write to any database (for developers)
  - Read any database (for analysts)
  - Built-in role: Atlas admin (for team leads)
Restrict Access to Specific Clusters: [Optional]
    4. Click Add User

    Step 3: Configure Network Access¶

    1. Navigate to Network Access
    2. Click Add IP Address
    3. Choose appropriate option:

    For specific IP: 

    Access List Entry: 192.168.1.100/32
Comment: John Doe - Developer machine

    For office network: 

    Access List Entry: 192.168.1.0/24
Comment: Office network range

    For temporary access (not recommended for production): 

    Access List Entry: 0.0.0.0/0
Comment: Allow access from anywhere (temporary)

    1. Click Confirm

    Step 4: Generate Connection String¶

    1. Navigate to Clusters
    2. Click Connect on your cluster
    3. Choose Connect your application
    4. Select driver and version
    5. Copy the connection string
    6. Replace <username> and <password> with colleague's credentials

    Example: 

    mongodb+srv://john_doe:securePassword123@democluster.c5lpr.mongodb.net/events_db?retryWrites=true&w=majority

    Using the API Setup Script¶

    # atlas_setup.py - Enhanced version for colleagues
from atlas_setup import AtlasAPIConnector
import json

def setup_colleague_access(colleague_info):
    """
    Set up database access for a colleague using Atlas API

    Args:
        colleague_info (dict): Colleague information
    """

    # Initialize API connector
    connector = AtlasAPIConnector()

    print(f"🔧 Setting up access for {colleague_info['name']}...")

    try:
        # 1. Create database user
        username = colleague_info['email'].split('@')[0].replace('.', '_')
        password = connector.generate_password()

        user_response = connector.create_database_user(
            username=username,
            password=password,
            roles=[{"roleName": "readWrite", "databaseName": "events_db"}]
        )

        if user_response:
            print(f"✅ Database user '{username}' created successfully!")

        # 2. Add IP to access list
        ip_response = connector.add_ip_to_access_list(
            ip_address=colleague_info['ip_address'],
            comment=f"{colleague_info['name']} - {colleague_info['role']}"
        )

        if ip_response:
            print(f"✅ IP address {colleague_info['ip_address']} added to access list!")

        # 3. Generate connection string
        connection_string = connector.get_connection_string(username, password)

        # 4. Save colleague information
        colleague_access = {
            "name": colleague_info['name'],
            "email": colleague_info['email'],
            "username": username,
            "password": password,
            "connection_string": connection_string,
            "setup_date": datetime.now().isoformat(),
            "access_level": colleague_info.get('access_level', 'readWrite')
        }

        # Save to file for record keeping
        with open(f"access_records/{username}_access.json", "w") as f:
            json.dump(colleague_access, f, indent=2)

        print(f"📝 Access information saved to access_records/{username}_access.json")
        print(f"🔗 Connection string: {connection_string}")

        return colleague_access

    except Exception as e:
        print(f"❌ Setup failed: {e}")
        return None

# Example usage
if __name__ == "__main__":
    colleague_data = {
        "name": "Jane Smith",
        "email": "jane.smith@company.com",
        "ip_address": "203.0.113.45",
        "role": "Data Analyst",
        "access_level": "read"
    }

    result = setup_colleague_access(colleague_data)
    if result:
        print("🎉 Colleague access setup complete!")

    Batch Colleague Setup¶

    For setting up multiple colleagues at once:

    # batch_setup.py
import csv
import json
from datetime import datetime

def batch_setup_colleagues(csv_file_path):
    """
    Set up database access for multiple colleagues from CSV file

    CSV format:
    name,email,ip_address,role,access_level
    John Doe,john@company.com,192.168.1.100,Developer,readWrite
    Jane Smith,jane@company.com,192.168.1.101,Analyst,read
    """

    results = []

    with open(csv_file_path, 'r') as file:
        reader = csv.DictReader(file)

        for row in reader:
            print(f"\n📋 Processing: {row['name']}")

            result = setup_colleague_access(row)
            if result:
                results.append(result)
            else:
                print(f"❌ Failed to setup access for {row['name']}")

    # Generate summary report
    summary = {
        "setup_date": datetime.now().isoformat(),
        "total_colleagues": len(results),
        "successful_setups": len([r for r in results if r]),
        "colleagues": results
    }

    with open("batch_setup_summary.json", "w") as f:
        json.dump(summary, f, indent=2)

    print(f"\n📊 Batch setup complete! {len(results)} colleagues configured.")
    print("📝 Summary saved to batch_setup_summary.json")

    Password Security¶

    Strong Password Guidelines: - Minimum 12 characters - Mix of uppercase, lowercase, numbers, symbols - No personal information - Unique for each service

    Password Generation: 

    import secrets
import string

def generate_secure_password(length=16):
    """Generate a cryptographically secure password"""

    characters = (
        string.ascii_letters + 
        string.digits + 
        "!@#$%^&*"
    )

    password = ''.join(
        secrets.choice(characters) 
        for _ in range(length)
    )

    return password

# Example usage
new_password = generate_secure_password()
print(f"Generated password: {new_password}")

    Tracking Colleague Access¶

    Create a simple tracking system:

    # access_tracker.py
import json
from datetime import datetime
from pathlib import Path

class AccessTracker:
    def __init__(self, tracking_file="colleague_access_log.json"):
        self.tracking_file = Path(tracking_file)
        self.load_access_log()

    def load_access_log(self):
        """Load existing access log or create new one"""
        if self.tracking_file.exists():
            with open(self.tracking_file, 'r') as f:
                self.access_log = json.load(f)
        else:
            self.access_log = {
                "created": datetime.now().isoformat(),
                "colleagues": {}
            }

    def add_colleague_access(self, colleague_info):
        """Add new colleague access record"""
        username = colleague_info['username']

        self.access_log['colleagues'][username] = {
            "name": colleague_info['name'],
            "email": colleague_info['email'],
            "setup_date": datetime.now().isoformat(),
            "access_level": colleague_info['access_level'],
            "ip_address": colleague_info['ip_address'],
            "status": "active"
        }

        self.save_access_log()

    def revoke_colleague_access(self, username):
        """Mark colleague access as revoked"""
        if username in self.access_log['colleagues']:
            self.access_log['colleagues'][username]['status'] = 'revoked'
            self.access_log['colleagues'][username]['revoked_date'] = datetime.now().isoformat()
            self.save_access_log()

    def save_access_log(self):
        """Save access log to file"""
        with open(self.tracking_file, 'w') as f:
            json.dump(self.access_log, f, indent=2)

    def generate_access_report(self):
        """Generate a report of all colleague access"""
        colleagues = self.access_log['colleagues']

        active_count = len([c for c in colleagues.values() if c['status'] == 'active'])
        revoked_count = len([c for c in colleagues.values() if c['status'] == 'revoked'])

        print("📊 Colleague Access Report")
        print("=" * 30)
        print(f"Total colleagues: {len(colleagues)}")
        print(f"Active access: {active_count}")
        print(f"Revoked access: {revoked_count}")
        print("\n👥 Active Colleagues:")

        for username, info in colleagues.items():
            if info['status'] == 'active':
                print(f"  • {info['name']} ({username}) - {info['access_level']}")

# Example usage
tracker = AccessTracker()
tracker.generate_access_report()

    Verification Script¶

    # test_colleague_access.py
from mongo_connector import MongoConnector
import os

def test_colleague_connection(connection_string):
    """
    Test if colleague can connect with their credentials

    Args:
        connection_string (str): Full MongoDB connection string
    """

    print("🧪 Testing colleague database access...")

    try:
        # Extract password from connection string for MongoConnector
        # Format: mongodb+srv://username:password@cluster.../
        parts = connection_string.split('@')[0].split(':')
        password = parts[-1]

        # Initialize connection
        mongo = MongoConnector(password)

        # Test basic operations
        print("✅ Connection established")

        # Test read operation
        count = mongo.count_events()
        print(f"📊 Can read events: {count} events found")

        # Test write operation (if allowed)
        try:
            test_event = {
                "id": "colleague_test_001",
                "title": "Colleague Access Test",
                "location": "Test Location",
                "source": "access_test"
            }

            result = mongo.insert_event(test_event)
            if result:
                print("✅ Can write events")

                # Clean up test event
                mongo.delete_event({"id": "colleague_test_001"})
                print("🧹 Test event cleaned up")

        except Exception as write_error:
            print(f"⚠️ Write access limited: {write_error}")

        mongo.close()
        print("🎉 Colleague access test completed successfully!")
        return True

    except Exception as e:
        print(f"❌ Access test failed: {e}")
        return False

def test_all_colleagues():
    """Test access for all configured colleagues"""

    access_records_dir = Path("access_records")

    if not access_records_dir.exists():
        print("📂 No access records found")
        return

    for record_file in access_records_dir.glob("*_access.json"):
        with open(record_file, 'r') as f:
            colleague_info = json.load(f)

        print(f"\n🧪 Testing access for {colleague_info['name']}...")
        test_colleague_connection(colleague_info['connection_string'])

    🆘 Troubleshooting¶

    Common Issues¶

    Issue: "Authentication failed" 

    Solution:
1. Check username/password spelling
2. Verify user exists in Database Access
3. Check user has correct permissions
4. Try regenerating password

    Issue: "IP not whitelisted" 

    Solution:
1. Check Network Access list
2. Verify IP address format
3. Add 0.0.0.0/0 for testing (temporary)
4. Check for dynamic IP changes

    Issue: "Connection timeout" 

    Solution:
1. Check internet connectivity
2. Verify firewall settings
3. Try different network
4. Check MongoDB Atlas status
